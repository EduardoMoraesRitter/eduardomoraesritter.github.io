=== CONVERSATIONAL ARCHITECTURE ===

The site displays an interactive diagram called "Conversational Architecture" representing how Eduardo designs AI systems in production. The components are:

- **Channels**: Unified message input — WhatsApp, Instagram, Webchat, and other omnichannel sources.
- **Queue (Communication Tunnel)**: Layer that receives, validates, and queues messages before processing. Uses patterns like API Gateway, SQS, and Rate Limiting for resilience.
- **AI Engine**: Intelligence core — the LLM (e.g., Claude 3.5) that processes and reasons about the message.
- **Knowledge Base**: Knowledge base with semantic search via RAG (Retrieval-Augmented Generation), using Vector DBs like Pinecone.
- **Tools**: Skills and external connections — APIs, functions, and integrations that expand agent capabilities.
- **Prompts**: Repository of templates and optimized System Prompts using Prompt Engineering techniques and personas.
- **Observability (Monitoring)**: Real-time monitoring dashboard with latency metrics (e.g., 24ms), tokens per second (e.g., 85.2), and success rate (e.g., 99.9%).

**Architecture flow:**
1. Message arrives via Channels → enters the Queue
2. Queue processes and routes to AI Engine
3. AI Engine retrieves context from Knowledge Base (RAG)
4. AI Engine executes Tools if needed
5. Response returns via the same path to the user
6. Logs and metrics feed the Observability layer
7. Observability feeds back into Prompts with continuous evolution data
